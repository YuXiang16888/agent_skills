# 專業 Email 撰寫指南

## 核心原則

語氣定位：**專業但有溫度**。不過度正式（避免讓人感覺距離很遠），也不過於隨意（避免不夠尊重）。像是一位認真、親切又有條理的同事寫的信。

---

## Email 結構模板

### 1. 主旨（Subject）
- 格式：`[動作標籤] 核心內容 — 補充說明（選填）`
- 範例：
  - `[活動邀請] 2025 產品發表會 — 3/15 台北信義區`
  - `[請求確認] 合約文件簽署進度`
  - `[提醒] 本週五截止：問卷填寫`
  - `[產品詢問] OO 方案的報價與規格`
- 原則：一眼看懂目的、能判斷優先順序、避免模糊詞如「關於...」「請問...」

---

### 2. 稱謂
| 關係 | 寫法 |
|------|------|
| 不熟悉 / 第一次聯繫 | `您好，XX 先生 / 女士：` |
| 有往來但不熟 | `XX 您好：` |
| 同事 / 熟識 | `Hi XX，` |
| 多人 | `各位好，` / `Hi All，` |

---

### 3. 開場白（第一段）
直接說明聯繫目的，不要讓對方猜。避免空泛的「希望您一切都好」（除非有真實脈絡）。

✅ 好的開場：
- `這封信是想邀請您參加我們即將舉辦的...`
- `想跟您確認一下上週討論的...`
- `冒昧打擾，想詢問關於...`

❌ 避免：
- `我是 OO 公司的 XX，非常高興能聯繫到您...`（太冗）
- `您好，不知道您最近是否方便...`（太迂迴）

---

### 4. 主體內容
- **一封信只講一件事**，若有多項請求，分點條列
- 條列格式：簡短說明 + 若有需要對方確認的，用 `→` 標示行動
- 重要資訊（日期、地點、截止時間）**加粗**
- 段落之間空行，不要一大段文字

---

### 5. 明確的行動呼籲（Call to Action）
告訴對方「你希望他做什麼」以及「什麼時候前完成」。

✅ 範例：
- `麻煩您於 **3/10 前** 回覆是否能出席，感謝！`
- `若有任何問題，歡迎直接回信或來電，我們會盡快回覆。`
- `請確認以上內容是否正確，若有需要修改的地方請告知。`

---

### 6. 結尾敬語

| 情境 | 結尾語 |
|------|--------|
| 正式商務 | `敬祝 商祺` / `順頌時祺` |
| 一般往來 | `謝謝您的協助` / `感謝您的配合` |
| 內部同事 | `謝謝！` / `感謝 🙏` |
| 英文結尾 | `Best regards,` / `Thanks,` |

署名建議包含：姓名、職稱、公司、聯絡方式（電話 / 信箱）

## 各情境撰寫重點

### 📅 活動邀請
- 清楚列出時間、地點、主題
- 說明「有什麼亮點」讓對方有參加動機
- 附上報名連結與截止日期
- 語氣可稍微熱情一點，帶點期待感

### 📄 文件申請
- 說明申請原因（簡短即可）
- 列出需要的文件清單
- 說明用途（若對方可能有疑慮）
- 給對方一個合理的回覆期限

### ⏰ 事件提醒
- 主旨直接點出「提醒」
- 重複關鍵資訊（截止日期、地點等）
- 語氣要提醒但不催促，友善為主
- 若是第二次提醒，可稍微說明「因為重要所以再提醒一次」

### 🛍 產品詢問
- 先簡單介紹自己公司背景（1-2 句即可）
- 明確說明想了解的項目（規格、報價、試用等）
- 若有數量或時程需求，一起說明
- 結尾表示期待進一步溝通

## 常見錯誤 ❌

| 錯誤 | 改法 |
|------|------|
| 主旨太模糊「請問一下」 | 改成具體標籤 + 內容 |
| 開頭沒有目的，繞了一大圈 | 第一句直接說為什麼寫這封信 |
| 沒有說清楚希望對方做什麼 | 明確寫出 CTA（行動呼籲） |
| 附件沒有提及或命名不清 | 正文中說「請見附件：OO文件_v1」 |
| 一封信塞了太多請求 | 分拆成多封信，或條列清楚 |
| 語氣太過正式像法律文件 | 加入一句親切的說明或謝語 |
| 忘記抄送（CC）相關人員 | 寄出前確認收件人與 CC 名單 |
